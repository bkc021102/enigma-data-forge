Deep-dive highlights

Flight Delay Prediction & Analytics System

Apache Spark PySpark ML Pipelines MCP

Built an end-to-end system that takes raw aviation + weather + turbulence data, converts it into ML-ready features, trains predictive models, and exposes insights through an AI-queryable interface.

  • Multi-source integration: Joined aviation records with weather/turbulence/visibility signals and handled missing values, inconsistent timestamps, and schema differences using Spark transformations.
  • Feature engineering: Created delay-risk indicators (temporal patterns, route context, severity signals, airport-level factors) to build clean training tables for modeling.
  • Model training + tuning: Trained Spark MLlib models (Logistic Regression, Random Forest, Gradient-Boosted Trees) using pipeline workflows and cross-validation with consistent evaluation.
  • Usable outputs: Produced structured prediction outputs and summary views so results can be reused in reporting, analysis, or a downstream application.
  • AI-queryable analytics: Designed an MCP-based interface so a user can ask natural-language questions and get answers backed by your Spark outputs.
Shows: scalable data processing + ML workflow thinking End-to-end system design Production-style structure

Healthcare Admissions Database System

PostgreSQL dbt Docker Streamlit

Designed a full analytics-ready healthcare data platform — from raw CSV ingestion to validated OLAP models and an interactive dashboard.

  • Clean ingestion: Built Python ETL validations (types, null checks, and consistency rules) before loading to PostgreSQL.
  • Strong data modeling: Designed a normalized 3NF OLTP schema for data integrity, then created an analytics-ready star schema with dbt (fact + dimensions).
  • Advanced SQL insights: Used CTEs and window functions to build operational analytics (e.g., appointment patterns/no-show style analysis).
  • Delivery layer: Built a Streamlit dashboard for non-technical users to explore KPIs without writing SQL.
  • Reproducible deployment: Containerized the stack with Docker for consistent setup and execution.
Shows: analytics engineering maturity OLTP → OLAP thinking

Cybersecurity Data Lakehouse (Medallion)

Databricks PySpark Delta Lake Medallion

Built a lakehouse pipeline for cybersecurity JSON data, focusing on scalable transformations and analytics-ready structure.

  • Bronze/Silver pipeline: Implemented medallion stages to move from raw JSON ingestion → cleaned and structured analytics tables.
  • Complex JSON normalization: Flattened nested arrays/structures using Spark explode patterns to create granular vendor/product-level views.
  • Analytics foundation: Produced curated tables enabling vendor risk analysis and downstream reporting.
  • Engineering focus: Emphasized schema consistency, reliability, and reusability within Databricks + Delta Lake patterns.
Shows: lakehouse + semi-structured data skill Big data transformations
